Manual Admin Setup Instructions

Since the automatic setup tools are having permission issues, follow these manual steps to set up your first admin user.

‚ö†Ô∏è Important: You need to have access to the Firebase Console for this project. If you don't have access, contact the project owner.

Method 1: Using Firebase Console (Recommended)

Step 1: Create a Regular User Account

First, create a regular user account through the app:

  1. Go to https://rasin.pyebwa.com/app/
  2. Sign up or log in with your email
  3. Complete the onboarding process
  4. Note your email address - you'll need it for the next steps

Step 2: Access Firebase Console

Open the Firebase Console for this project:

  1. Go to Firebase Console - Firestore
  2. Sign in with your Google account (must have project access)
  3. Navigate to Firestore Database ‚Üí users collection

Step 3: Find Your User Document

In the users collection:

  1. Look for the document with your email address
  2. Click on the document to open it
  3. You should see fields like email, createdAt, etc.

Step 4: Add Admin Fields

Click the "Add field" button and add these fields:

  • Field: role
    Type: string
    Value: superadmin (or admin or moderator)
  • Field: isAdmin
    Type: boolean
    Value: true
  • Field: adminPromotedAt
    Type: timestamp
    Value: Click "Add timestamp" button
  • Field: adminPromotedBy
    Type: string
    Value: manual-setup

Click Save after adding all fields.

‚úÖ Success! You've now promoted your user to admin. You can access the admin dashboard at https://rasin.pyebwa.com/app/admin

Method 2: Using Firebase CLI (Advanced)

If you have Firebase CLI access:

# 1. Install Firebase CLI npm install -g firebase-tools # 2. Login to Firebase firebase login # 3. Connect to your project firebase use pyebwa-f5960 # 4. Run Firestore shell firebase firestore:shell # 5. Update your user document (replace YOUR_USER_ID) db.collection('users').doc('YOUR_USER_ID').update({ role: 'superadmin', isAdmin: true, adminPromotedAt: firebase.firestore.FieldValue.serverTimestamp(), adminPromotedBy: 'cli-setup' });

Role Descriptions

  • superadmin: Full access to all admin features including system settings
  • admin: Can manage users, family trees, and content
  • moderator: Can only manage content (limited access)
